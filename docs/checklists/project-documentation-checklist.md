# Checklist de Documentação de Projeto

Use esta checklist sempre que terminar um projeto ou uma entrega relevante.

## Base do case

- [ ] Defini o nome do projeto e o `slug`
- [ ] Registrei o contexto do projeto
- [ ] Expliquei o problema de negócio
- [ ] Listei os objetivos principais
- [ ] Deixei clara a minha atuação

## Conteúdo para o portfólio

- [ ] Escrevi um `eyebrow` curto e estratégico
- [ ] Escrevi uma descrição curta com contexto, solução e impacto
- [ ] Escolhi exatamente 3 highlights
- [ ] Listei a stack principal
- [ ] Identifiquei o que esse projeto comprova sobre meu perfil técnico

## Conteúdo técnico

- [ ] Documentei a arquitetura ou o fluxo principal
- [ ] Registrei integrações, filas, jobs, cache ou webhooks quando existirem
- [ ] Listei regras de negócio importantes
- [ ] Anotei decisões técnicas e trade-offs
- [ ] Registrei desafios reais do projeto
- [ ] Documentei como os desafios foram resolvidos

## Resultado e impacto

- [ ] Registrei métricas ou ganhos operacionais
- [ ] Anotei melhoria de performance, confiabilidade ou produtividade
- [ ] Registrei aprendizados
- [ ] Listei próximos passos, se fizer sentido

## Material de apoio

- [ ] Separei links públicos, se existirem
- [ ] Separei prints ou diagramas sem expor dados sensíveis
- [ ] Revisei informações confidenciais
- [ ] Deixei uma versão curta pronta para virar card no portfólio
